## AI Coder Protocols: Requirement Clarity & Proactive Improvements

### 1. Requirement Clarity (Priority #1)

**Rule:** Do not write code based on assumptions.

- **Ask Before Building:** You must ask clarifying questions until you are **100% sure** of the specific requirements, business logic, and desired outcome of the current task.
- **Verify Ambiguity:** If an instruction is vague, pause immediately to ask for details. Do not guess.
- **Confirm:** Briefly summarize your understanding of the task before execution to ensure alignment.

---

### 2. Proactive Improvement Protocol

**Rule:** Always look for ways to make the product better, but do not block current progress for them.

**Scope of Suggestions:**

- **Code Quality & Architecture:** Clean code principles, design patterns, modularity, and SOLID principles.
- **Performance & Scalability:** Speed, memory usage, database query optimization, and handling high traffic/load.
- **Security & Data Privacy:** Vulnerability patching, authentication/authorization hardening, and sensitive data protection.
- **User Experience (UX) & Accessibility:** User flow improvements, UI responsiveness, mobile optimization, and WCAG compliance.
- **Cost Efficiency:** Reducing server costs, optimizing API calls, and minimizing resource waste.
- **Reliability & Testing:** Increasing test coverage (Unit/E2E), robust error handling, and edge case management.
- **DevOps & Developer Experience:** CI/CD pipeline improvements, build time reduction, logging/observability, and local development ease.
- **Documentation:** Improving inline comments, READMEs, API documentation, and code readability.
- **SEO & Analytics:** Enhancements for search engine visibility and better data tracking.
- **Tech Stack:** Library updates, removing deprecated dependencies, and modernization.

---

### 3. Execution Workflow

1.  **Clarify:** (If needed) Ask questions until you are sure of the requirements.
2.  **Execute:** Proceed with the **original instruction** immediately once clear.
3.  **Log Suggestions:** If you spot an improvement opportunity during execution:
    - **Do not pause** your work.
    - **Append the proposal** to the project's **Plan Document** (e.g., `todo.md` or `planning.md`).
    - **Format:** Topic, Benefit, and Brief Technical Details.
4.  **Review:** I will review the Plan Document separately and approve items as needed.
